### **MSBUILD : warning CA0062 : The rule 'CA1825' referenced in rule set was not found**
この警告が発生する主な原因と対策を説明します。

#### **原因**
1. **CA1825ルールが適切なアナライザーに含まれていない**
   - `CA1825` は `Microsoft.CodeAnalysis.NetAnalyzers` に含まれる静的解析ルールですが、Visual Studio 2017 (15.9.68) ではデフォルトで提供されていない可能性があります。

2. **ルールセットファイル (.ruleset) のパスやフォーマットに問題がある**
   - `CustomRules.ruleset` が `\\Cminfile001\common\Run\CustomRuleSet\CustomRules.ruleset` というネットワークパスに存在すると仮定すると、アクセス権の問題や、MSBuild で正しく認識されない問題が考えられます。

3. **MSBuildのバージョンと.NET Frameworkの互換性**
   - `MSBuild 15.9.21` を使用しており、.NET Framework 4.7.2 がターゲットになっています。新しい.NET Analyzerのルールが適用されない可能性があります。

4. **FxCopと.NET Code Analysis (Roslyn Analyzers) の混在**
   - Visual Studio 2017 では `FxCop` に依存する解析ルールがあり、`CA1825` が `FxCopAnalyzers` ではなく `Microsoft.CodeAnalysis.NetAnalyzers` に移行された可能性があります。

#### **対策**
1. **NuGet パッケージを明示的に追加**
   - `.csproj` に以下のパッケージを追加して `CA1825` を含む解析ルールを適用する。

   ```xml
   <ItemGroup>
       <PackageReference Include="Microsoft.CodeAnalysis.NetAnalyzers" Version="8.0.0" PrivateAssets="all" />
   </ItemGroup>
   ```

2. **.ruleset ファイルのパスを確認**
   - `CustomRules.ruleset` のパスがネットワーク経由になっている (`\\Cminfile001\common\Run\CustomRuleSet\CustomRules.ruleset`) ため、ローカルにコピーして参照する。

   ```xml
   <CodeAnalysisRuleSet>$(ProjectDir)\CustomRules.ruleset</CodeAnalysisRuleSet>
   ```

3. **MSBuild のバージョンを確認**
   - `MSBuild 15.9.21` を使用していますが、`Microsoft.CodeAnalysis.NetAnalyzers` の一部のルールは `MSBuild 16.0` 以降で動作が保証されています。
   - Visual Studio 2019 以降にアップグレードすることで解決する可能性があります。

4. **ルールセットファイルのフォーマットを確認**
   - `.ruleset` ファイル内で `CA1825` が正しく定義されているか確認。

   ```xml
   <Rule Id="CA1825" Action="Warning" />
   ```

   `Action="Warning"` ではなく `Action="None"` になっている場合、ルールが適用されない可能性があります。

5. **MSBuild コマンド実行時に `RunCodeAnalysis` を無効化**
   - ビルド時の `CodeAnalysisRuleSet` の参照エラーを回避するため、以下のように `RunCodeAnalysis=false` を指定してビルドする。

   ```sh
   msbuild /p:RunCodeAnalysis=false
   ```

6. **FxCop Analyzers を有効化**
   - Visual Studio 2017 では `FxCopAnalyzers` を追加することで、旧バージョンの `CA` ルールを利用できる可能性があります。

   ```xml
   <ItemGroup>
       <PackageReference Include="Microsoft.CodeAnalysis.FxCopAnalyzers" Version="2.9.8" />
   </ItemGroup>
   ```

#### **まとめ**
この問題の解決には、以下のステップを試してください：
1. `Microsoft.CodeAnalysis.NetAnalyzers` を `csproj` に追加
2. `.ruleset` のパスをローカルに変更
3. `MSBuild 16.0+` にアップグレードを検討
4. `.ruleset` の `CA1825` 設定を確認
5. `RunCodeAnalysis=false` で試験的に無効化してビルド
6. `FxCopAnalyzers` を試験的に追加して動作確認

問題が解決しない場合は、追加のログを確認しながら `Microsoft.CodeAnalysis` のバージョン依存関係を調査すると良いでしょう。

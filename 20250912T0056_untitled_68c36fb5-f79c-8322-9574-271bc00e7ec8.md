司令、以下は **Windows 11 / Microsoft 365 (Excel, Access を含む Office 環境)** に移行した際に注意すべき、廃止された機能・仕様変更・非推奨（deprecated）事項の一覧と、それぞれの代替手段の案です。必要なら Excel と Access を分けた詳細もできます。

---

## 廃止／仕様変更／非推奨となっている機能・注意点

| 製品 | 機能・仕様 | 状況・影響 | 非推奨・廃止時期等 |
|---|---|---|---|
| **Access** | **ADP (Access Data Projects)** | Access 2013 以降、ADP ファイルのサポートが削除されている。ADP を使っていたプロジェクトはそのまま開けない。 citeturn2search0turn2search6turn2search10 | 廃止済み（Access 2013 から） |
| Access | Jet 3.x IISAM（Access 97 など旧バージョンの DB 形式／ドライバ） | Access 2013 から、Access 97 のデータベースを直接開くことができない。変換が必要。 citeturn2search0 | 廃止済み（Access 2013） |
| Access | PivotCharts と PivotTables via Office Web Components | Office Web Components のサポートがなくなっており、Access 作成時にこれらが使えない。代替手段が必要。 citeturn2search0 | 廃止済み（Access 2013） |
| Access | Smart Tags / Action Tags | Smart Tags プロパティがサポートされなくなっている。 citeturn2search0 | 廃止済み（Access 2013） |
| Access | データ コレクション フォーム（Data Collection forms）を新規作成する機能 | Access 2013 以降、新規作成不可。従前作成されたものは処理されるが、将来的な互換性に注意。 citeturn2search0 | 非推奨/廃止傾向 |
| Excel / 全体 Office 環境 | “Excel 4.0 マクロ (XLM マクロ)” | 現在も実行可能だが、Microsoft はこれらマクロを最新の VBA に移行するよう推奨している。セキュリティや互換性の面で将来的に制限が強まる可能性あり。 citeturn2search3 | 非推奨としての警告あり |
| Excel | Visual Basic for Applications が無効にされた環境での機能制限 | VBA が無効なら、VBAプロジェクト、XLMマクロ、ActiveXコントロールといった機能が使えない。関連するアドインやツールも制限を受ける。 citeturn2search1 | 状況依存（ポリシー設定など） |
| Windows 11 / OS 側 | スタートメニューでアプリを右クリック → 最近使ったファイルや “固定されたファイル (Pinned)” の一覧が表示されなくなっている | Windows 10 では可能だったが、Windows 11 ではスタートメニューの右クリックで直接それが出ない変化あり。 citeturn0search9 | 仕様変更済み |

## 代替手段・対応策

以下は、上記の非推奨・廃止事項に対して取れる代替手段・改善策です。

| 廃止項目 | 対応策・代替案 |
|---|---|
| **ADP が使えない** | - ACCDB や MDB フォーマットを使い、SQL Server 等をバックエンドにして「リンクテーブル + フォーム／レポート／VBA」を使う設計に移行する。<br> - データアクセス／クエリ／ストアドプロシージャの部分を SQL Server 側に移して、Access はフロントエンドとしてフォーム・レポートのみ持つ。 |
| Jet / Access 97 形式サポート廃止 | - 古いデータベースは .accdb形式に変換する。<br> - 必要であれば、中間バージョン（Access 2010 等）を使って変換プロセスを挟む。 |
| PivotCharts / PivotTables via Office Web Components 廃止 | - Excel の最新機能を使ってチャートやピボットテーブルを作成し、必要なら Access 内から Excel を呼び出したり、Excel 経由でレポート出力する。<br> - Power BI や Excel のデータ可視化ツールを併用する。 |
| Smart Tags / データ コレクション フォーム廃止 | - 新しいフォームや UI を Access または別のアプリで構築する。<br> - データ収集に関しては、Microsoft Forms、Power Apps、SharePoint リストなどクラウドベースの代替を検討する。 |
| Excel 4.0 マクロ非推奨 | - VBA マクロへ移行する。XLM マクロでしかできない部分は、新しい VBA のオブジェクトモデルで代替できるか確認。<br> - 将来的には Power Query、Office Scripts、または Power Automate 等を使って自動化・処理を移行する可能性を検討。 |
| VBA が無効な環境での制限 | - セキュリティポリシーを確認して、VBA を適切に許可できるように設定を見直す。<br> - ActiveX を使っている部分は標準コントロールや他の UI 技術 (例えば WinForms, WPF, Power Apps など) に置き換える。 |
| Windows 11 のスタートメニューでの UI 変化 | - タスクバーにアプリをピン留めし、タスクバーアイコンから右クリックで最近使用したファイルを表示する。<br> - スタートメニューの設定を見直すか、サードパーティツール（Start メニューカスタマイズ）を使って以前の動きを模倣する。 |

---

もしよければ、**Excel と Access 各々**でより詳細な非推奨項目（マクロ、データタイプ、互換性、UI など）をまとめて提供しますか？また、司令が特に使っている機能（マクロ、外部データ接続、レポーティング等）を教えてもらえれば、それに特化した影響と代替案も提案できます。
